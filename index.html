Gitleaks: Deteção de Credenciais Expostas

O Gitleaks realiza varrimento estático no histórico de commits do Git e no código atual para identificar tokens de API, chaves privadas SSH, credenciais AWS, tokens JWT e palavras-passe que tenham sido acidentalmente incluídas no código fonte.

Verificação de alta velocidade com entropia de Shannon.
Prevenção de fuga de segredos antes do Push no GitHub.
Integrado como o primeiro estágodo pipeline CI/CD.

Ficheiro de Configuração (.gitleaks.toml):

title = "Gitleaks DevSecOps Config"

[[rules]]
id = "generic-api-key"
description = "Deteção de API Keys no código"
regex = '''(?i)(api[_-]?key|secret|token)\s*[:=]\s*["']([A-Za-z0-9_\-]{20,})["']'''
tags = ["key", "secret"]
Risco Mitigado: Fuga de Credenciais no repositório público (OWASP A07:2021-Identification and Authentication Failures).
Inseguro 
const AWS_SECRET = "AKIAIOSFODNN7EXAMPLE";
const dbPassword = "AdminPassword123!";
Seguro (SSDLC Best Practice) 
const AWS_SECRET = process.env.AWS_SECRET_KEY;
const dbPassword = process.env.DB_PASSWORD;

Semgrep: Análise Estática de Código (SAST)

O Semgrep é uma ferramenta ultrarrápida de análise estática (SAST) baseada em regras semânticas. Analisa a árvore de sintaxe abstrata (AST) do código para identificar antipadrões de segurança, injeções SQL, XSS, e maus usos de APIs sem a necessidade de compilar o projeto.

Exemplo de Regra Semgrep (.semgrep.yml):

rules:
  - id: detect-eval-usage
    pattern: eval(...)
    message: "Uso inseguro da função eval() detetado! Risco de RCE."
    severity: ERROR
Integração GitHub Security: O Semgrep gera relatórios no formato estandardizado .sarif, permitindo visualizar os achados diretamente no tab "Security" do repositório GitHub.

SonarQube: Quality Gate & Security Hotspots

O SonarQube combina análise de qualidade de código (Code Smells, Duplicações, Cobertura de Testes) com análise avançada de segurança (Security Vulnerabilities & Security Hotspots). No GitHub Actions, ele aplica o **Quality Gate**: se a aplicação contiver vulnerabilidades críticas, o pipeline é interrompido imediatamente.

Snyk: Software Composition Analysis (SCA)

A maior parte do código de uma aplicação moderna provém de bibliotecas e pacotes de terceiros (npm, pip, maven). O Snyk realiza a análise de composição de software (SCA), confrontando as dependências do package.json / package-lock.json com uma vasta base de dados de CVEs conhecidas.

Vantagens do Snyk no Pipeline CI/CD:

  • Deteção de vulnerabilidades diretas e transitivas.
  • Recomendação automática de atualizações de versão de pacotes (Auto PRs).
  • Bloqueio de dependências com licenças incompatíveis ou maliciosas.

Trivy: Varrimento e Hardening de Imagens de Containers

O Trivy (Aqua Security) é um scanner completo para containers Docker, sistemas de ficheiros e ficheiros de infraestrutura como código (IaC). Ele identifica CVEs no sistema operativo do container (ex: Alpine, Debian) e valida boas práticas de hardening no Dockerfile.

O Ciclo de Vida Seguro do Software (SSDLC)

1. Requisitos & Design

Modelação de Ameaças (Threat Modeling) e definição dos requisitos de segurança.

2. Desenvolvimento

Codificação segura, linters e deteção de credenciais via Gitleaks no pre-commit.

3. CI/CD Integration

Testes estáticos (Semgrep + SonarQube) e análise de dependências (SCA Snyk) no GitHub Actions.

4. Empacotamento

Hardening e varrimento de vulnerabilidades em containers com Trivy.

5. Produção & Monitorização

Deploy seguro condicionado pela aprovação rigorosa nos Quality Gates.
